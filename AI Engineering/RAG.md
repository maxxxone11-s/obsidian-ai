# RAG

Retrieval-Augmented Generation — поиск + генерация.

## Простое объяснение

RAG — это система, которая ищет релевантную информацию и используя её, генерирует ответ.

## Интуитивное объяснение

Как студент, который сначала ищет в книге информацию, а потом пишет ответ на основе найденного.

## Зачем это нужно

- Улучшает точность LLM
- Добавляет контекст из собственных данных
- Решает проблему hallucination

## Как это работает

### Этапы
1. Получить запрос пользователя
2. Найти релевантные документы
3. Передать в LLM с контекстом
4. Получить ответ на основе контекста

## Пример

```python
# Псевдокод
query = "Что такое RAG?"
documents = retriever.search(query)
prompt = f"На основе: {documents}\nОтветь на: {query}"
answer = llm.generate(prompt)
```

## Типичные ошибки

- Плохой поиск релевантных документов
- Шум в контексте
- Слишком много документов
