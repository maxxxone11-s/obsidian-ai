# Root Parent Child Leaf

## Простое объяснение

Root — верхушка. Parent — родитель. Child — потомок. Leaf — вершина без детей.

## Зачем это нужно

Эти термины используются во всех древовидных структурах.

## Как это работает

Root обычно один. Parent имеет child-вершины. Leaf не имеет потомков и находится на конце ветки.

## Пример

В файловой системе корневая папка — root, вложенная папка — child, файл без потомков можно мыслить как leaf.

## Типичные ошибки

- Путать leaf с любой нижней вершиной, не проверяя наличие потомков.

## Ментальная модель

Любое дерево можно читать сверху вниз.
