# Список смежности

## Простое объяснение

Список смежности — один из самых распространенных способов хранения графа в памяти. Для каждой вершины хранится список ее соседей.

## Зачем это нужно

Именно этот способ хранения используется в большинстве графовых алгоритмов и библиотек, потому что он экономит память для разреженных графов.

## Как это работает

Каждая вершина является ключом, а значением является список смежных вершин. Сам список смежности не делает граф ориентированным или неориентированным: это определяется смыслом ребер и тем, как записаны соседства.

## Пример

```text
A: B, C
B: A
C: A, D
D: C
```

## Типичные ошибки

- Воспринимать список смежности как ориентированный граф по умолчанию.
- Думать, что способ хранения сам определяет смысл ребер.

## Вопросы для проверки

- Что хранит список смежности?
- Почему он экономит память?
- Почему он не определяет тип графа самостоятельно?
