# Что такое граф

## Простое объяснение

Если есть объекты и известно, как они связаны друг с другом, значит эти данные можно представить графом. Сам рисунок не является графом: он только визуализирует уже существующую структуру.

## Зачем это нужно

Большинство современных программных систем можно описать графами: Git, Docker, зависимости Python, файловую систему, LangGraph и вычислительный граф PyTorch.

## Как это работает

Граф существует только тогда, когда одновременно есть объекты и явно определенные связи между ними. Связи нельзя придумывать самостоятельно: нужно брать только то, что задано смыслом данных.

## Пример

- Пользователи и дружба.
- Python modules и imports.
- Git commits.
- Docker Container -> Volume.

## Типичные ошибки

- Попытка самостоятельно придумать связи между объектами.
- Считать рисунок самим графом.

## Ментальная модель

Граф — это не картинка, а модель объектов и связей. Не искать связи, а использовать только явно заданные.
