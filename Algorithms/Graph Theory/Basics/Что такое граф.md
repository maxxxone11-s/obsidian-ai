# Что такое граф

## Простое объяснение
Если есть объекты и известно, как они связаны между собой, это можно представить графом.

## Зачем это нужно
Практически все современные программные системы можно представить в виде графа: Git, Docker, файловую систему, зависимости Python, PyTorch, базы данных, LangGraph.

## Как это работает
Граф состоит из vertices/nodes и edges. Важно использовать только связи, которые явно заданы в данных, а не придумывать их самостоятельно. Один и тот же граф можно нарисовать разными способами.

## Пример
Пользователи и дружба, Git commits, Python imports, Docker, социальные сети.

## Типичные ошибки
- Попытка самостоятельно придумать связь между объектами.

## Ментальная модель
Граф — это не картинка, а модель объектов и связей. Не искать связи, а использовать только явно заданные.
