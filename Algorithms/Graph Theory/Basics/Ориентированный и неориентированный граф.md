# Ориентированный и неориентированный граф

## Простое объяснение
Главный вопрос: если A связан с B, означает ли это автоматически, что B связан с A?

## Зачем это нужно
Большинство систем программирования являются ориентированными графами.

## Как это работает
Тип графа определяется смыслом связи, а не только стрелкой на рисунке. Friendship обычно undirected, Instagram follows, Git, Docker и Python imports — directed.

## Пример
Git, Docker, FastAPI, PyTorch, Instagram.

## Типичные ошибки
- Один раз был перепутан тип графа для Python imports.

## Ментальная модель
Не спрашивать “есть ли стрелка?”, а спрашивать “может ли связь работать обратно?”.
