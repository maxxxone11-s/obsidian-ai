# Дерево как рекурсивная структура данных

## Кратко

Дерево состоит из узлов, каждый из которых хранит собственное значение и ссылки на дочерние узлы.

## Простое объяснение

Каждый ребенок сам является корнем своего собственного поддерева.

## Зачем это нужно

Большинство рекурсивных алгоритмов работают именно с такими структурами.

## Как это работает

Алгоритм получает один узел. Далее он работает только с этим узлом и его детьми, не зная размер всего дерева.

Это делает дерево естественной рекурсивной структурой: задача для дерева раскладывается на такие же задачи для его поддеревьев.

## Пример

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.children = []
```

## Типичные ошибки

- Ожидать, что узел знает своих родителей.
- Ожидать, что функция знает всю структуру дерева.

## Вопросы для проверки

- Почему каждого ребенка можно считать новым деревом?
- Почему функция принимает только один `node`?
