# Поиск максимальной глубины дерева

## Кратко

Максимальная глубина дерева определяется через максимальную глубину его поддеревьев.

## Простое объяснение

Каждый узел спрашивает глубину всех детей, выбирает максимальную и добавляет себя.

## Зачем это нужно

Это один из наиболее распространенных шаблонов рекурсии на собеседованиях и LeetCode.

## Как это работает

Если детей нет, глубина равна `1`. Иначе:

```text
depth = 1 + max(depth(child))
```

Здесь используется `max()`, потому что глубина — это самый длинный путь вниз, а не сумма всех узлов.

## Пример

```python
def max_depth(node):
    if not node.children:
        return 1
    return 1 + max(
        max_depth(child)
        for child in node.children
    )
```

## Типичные ошибки

- Двойной подсчет текущего узла.
- Использовать цикл одновременно с генератором `max()`.
- Добавлять `+1` дважды.
- Путать задачи на сумму и задачи на максимум.

## Вопросы для проверки

- Почему используется `max()`, а не сумма?
- Почему `+1` добавляется только один раз?
- Что возвращает лист?
