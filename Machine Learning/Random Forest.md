# Random Forest

Ансамбль решающих деревьев. Не требует масштабирования, так как основан на деревьях.

## Импорт KNOWLEDGE_EXPORT — Machine Learning Fundamentals — 2026-06-26

**Кратко:** Decision Tree принимает решения через последовательность условий. Random Forest объединяет множество независимых деревьев. XGBoost строит деревья последовательно, исправляя ошибки предыдущих.

**Простое объяснение:** Одно дерево → голосование деревьев → обучение на ошибках предыдущих деревьев.

**Зачем это нужно:** Это самые популярные алгоритмы классического ML для табличных данных.

**Как это работает:** Decision Tree строит правила. Random Forest обучает много деревьев независимо и агрегирует ответы. XGBoost добавляет деревья последовательно, каждое следующее исправляет ошибки предыдущих.

**Пример:** Decision Tree → Random Forest → XGBoost.

**Типичные ошибки:**
- Изначально путалась связь между типом задачи и алгоритмом.

## Простое объяснение
Random Forest объединяет много деревьев решений и агрегирует их ответы.

## Зачем это нужно
Ансамбль деревьев обычно стабильнее одного дерева и меньше переобучается.

## Как это работает
Много деревьев обучаются на разных подвыборках данных и признаков, затем голосуют или усредняют результат.

## Пример
100 деревьев голосуют за класс: большинство выбрало “спам”.

## Типичные ошибки
- Путать Random Forest и XGBoost.
- Думать, что Random Forest строит деревья последовательно.
