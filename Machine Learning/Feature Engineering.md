# Feature Engineering

## Простое объяснение
Иногда один хороший Feature лучше сотни случайных.

## Зачем это нужно
Часто сильнее влияет на качество модели, чем выбор алгоритма.

## Как это работает
Из сырых данных создают признаки, которые лучше выражают закономерности: отношения, агрегаты, категории, даты или domain-specific признаки.

## Пример
Средний чек = выручка / заказы.

## Типичные ошибки
- Недооценка качества признаков.
