# Feature Scaling

## Простое объяснение

Feature Scaling — это приведение признаков (features) к сопоставимому масштабу. Это нужно, чтобы все признаки "говорили на одном языке" и не доминировал тот, у которого просто большие числа.

## Интуитивное объяснение

Представьте, что вы сравниваете квартиры по двум признакам: площадь (100-200 кв.м) и цена (1-10 млн руб). Если просто применить алгоритм на эти числа, цена будет полностью доминировать, потому что её числа намного больше. Scaling — это как перевести оба признака в один стандартный масштаб.

**Аналогия:** 
> Как при сравнении роста (в см) и веса (в кг) нужно привести их к одним единицам измерения.

## Зачем это нужно

- **Ускоряет обучение моделей** — градиентный спуск работает быстрее
- **Помогает алгоритмам, чувствительным к масштабу** — KNN, SVM, Neural Networks
- **Делает признаки сопоставимыми** — один признак не доминирует над другими
- **Улучшает сходимость** — для методов оптимизации на основе градиента

## Как это работает

### Normalization (Минмакс масштабирование)

Приведение значений к диапазону [0, 1]:

$$X_{scaled} = \frac{X - X_{min}}{X_{max} - X_{min}}$$

- Минимальное значение становится 0
- Максимальное значение становится 1
- Все остальные значения распределяются между ними
- Сохраняет форму распределения

### Standardization (Z-score нормализация)

Преобразование признаков так, чтобы среднее было 0, а стандартное отклонение 1:

$$X_{scaled} = \frac{X - \mu}{\sigma}$$

Где:
- μ — среднее значение (mean)
- σ — стандартное отклонение (std deviation)

### Когда что использовать

| Метод | Когда использовать | Преимущества |
|-------|-------------------|-------------|
| **Normalization** | Когда вы знаете границы диапазона | Простота, фиксированный диапазон [0,1] |
| **Standardization** | Когда данные близки к нормальному распределению | Лучше для алгоритмов на основе расстояния |

## Пример

### Простой пример

```python
from sklearn.preprocessing import StandardScaler, MinMaxScaler
import numpy as np

# Исходные данные
X = np.array([[1, 2], [2, 3], [3, 4], [4, 5]])

# Normalization (MinMax)
scaler_minmax = MinMaxScaler()
X_normalized = scaler_minmax.fit_transform(X)
print("Нормализованные:\n", X_normalized)
# [[0.  0. ]
#  [0.33 0.33]
#  [0.67 0.67]
#  [1.  1. ]]

# Standardization
scaler_std = StandardScaler()
X_standardized = scaler_std.fit_transform(X)
print("Стандартизированные:\n", X_standardized)
# [[-1.46  -1.46]
#  [-0.49  -0.49]
#  [ 0.49   0.49]
#  [ 1.46   1.46]]
```

### Практический пример с реальной задачей

```python
from sklearn.preprocessing import StandardScaler
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.svm import SVC
import numpy as np

# Загружаем данные
iris = load_iris()
X, y = iris.data, iris.target

# Разделяем на train/test
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# ВАЖНО: Масштабируем
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # fit и transform на train
X_test_scaled = scaler.transform(X_test)       # только transform на test!

# Обучаем SVM (чувствительный к масштабу)
model = SVC()
model.fit(X_train_scaled, y_train)
accuracy = model.score(X_test_scaled, y_test)
print(f"Accuracy: {accuracy:.2f}")
```

## Типичные ошибки

1. **Забыли масштабировать тестовый набор**
   - ❌ Неправильно: `X_test = scaler.fit_transform(X_test)` (fit на тестовом наборе!)
   - ✅ Правильно: `X_test = scaler.transform(X_test)` (используем fit от train)
   - **Почему:** Тестовый набор должен масштабироваться на основе параметров train, чтобы симулировать реальное предсказание

2. **Масштабируют перед train/test split**
   - ❌ Неправильно: масштабируем все данные, потом делим
   - ✅ Правильно: делим сначала, потом масштабируем (на train данных)
   - **Почему:** Иначе информация из test набора "просочится" в train

3. **Не масштабируют для Decision Trees**
   - Это не критично, но и не повредит
   - Деревья работают на основе порогов, не на расстояниях

4. **Не знают, какой метод использовать**
   - Используйте **Standardization** как default
   - Normalization используйте, если вы уверены в диапазоне данных

## Когда масштабирование критично

Scaling необходим для:
- [[Linear Regression]] — использует градиентный спуск
- [[Logistic Regression]] — использует градиентный спуск
- [[KNN]] — основан на расстояниях между точками
- [[SVM]] — использует расстояния в feature space
- [[Neural Networks]] — градиентный спуск + multiple layers

Scaling почти не нужен для:
- [[Decision Tree]] — принимает решения через пороги (feature <= value)
- [[Random Forest]] — ансамбль деревьев
- [[XGBoost]] — ансамбль деревьев

## Почему деревьям не нужен scaling

**Decision Trees** принимают решения через пороги вида `feature <= value`.

Для дерева важен **порядок значений**, а не их **абсолютный масштаб**:

```
До scaling:          После scaling:
age <= 30?           age <= 0.5?
     |                    |
   Yes  No             Yes  No
   ...  ...             ...  ...
```

Порог просто сдвинется, но логика останется той же. Поэтому для деревьев масштабирование не критично.
