# Decision Tree

Алгоритм машинного обучения на основе дерева решений. Не требует масштабирования, так как принимает решения через пороги.

## Кратко

Decision Tree принимает решения через последовательность условий. Random Forest объединяет множество независимых деревьев. XGBoost строит деревья последовательно, исправляя ошибки предыдущих.

## Простое объяснение

Одно дерево → голосование деревьев → обучение на ошибках предыдущих деревьев.

Decision Tree принимает решение через последовательность вопросов и условий.

## Зачем это нужно

Это самые популярные алгоритмы классического ML для табличных данных.

Это один из самых понятных алгоритмов классического ML для табличных данных.

## Как это работает

Decision Tree строит правила. Random Forest обучает много деревьев независимо и агрегирует ответы. XGBoost добавляет деревья последовательно, каждое следующее исправляет ошибки предыдущих.

Дерево делит данные по признакам через пороги, пока не получает достаточно чистые группы для прогноза класса или числа.

## Пример

Decision Tree → Random Forest → XGBoost.

Если возраст <= 30, перейти в левую ветку, иначе — в правую.

## Типичные ошибки

- Изначально путалась связь между типом задачи и алгоритмом.

- Путать тип задачи Regression/Classification с алгоритмом.
- Думать, что деревьям обязательно нужен Feature Scaling.
