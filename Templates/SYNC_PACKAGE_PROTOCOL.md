# 📦 SYNC_PACKAGE Protocol

## Описание

**SYNC_PACKAGE Protocol** — это структурированный формат для обновления базы знаний AI Second Brain через текстовые пакеты данных. 

Протокол позволяет пользователю вставить одно сообщение, которое автоматически:
- Создаёт или обновляет концепцию
- Создаёт связанные stub-заметки
- Обновляет индексы
- Отслеживает прогресс обучения

---

## Формат SYNC_PACKAGE

### Текстовый формат (основной)

```
SYNC_PACKAGE:
topic: Название концепции
area: область-знания
status: draft|learning|needs_review|learned
confidence: 0.0-1.0
related: [Тема 1, Тема 2, Тема 3]
summary: Краткое резюме концепции
simple_explanation: Простое объяснение в 1-2 предложениях
examples:
  - Пример 1 с описанием
  - Пример 2 с описанием
common_mistakes:
  - Ошибка 1: Описание и решение
  - Ошибка 2: Описание и решение
review_questions:
  - Вопрос 1?
  - Вопрос 2?
```

### YAML Frontmatter для созданной заметки

```yaml
---
type: concept
area: [область]
status: [статус]
created: [дата]
updated: [дата]
tags: [теги]
confidence: [уровень]
last_review: [дата или null]
next_review: [дата или null]
---
```

## Обязательные поля

| Поле | Тип | Описание |
|------|-----|---------|
| `topic` | string | Название концепции (обязательно) |
| `area` | string | Область знания (обязательно) |
| `status` | enum | Статус изучения (обязательно) |
| `confidence` | float | Уровень уверенности 0-1 (обязательно) |
| `related` | array | Связанные концепции (обязательно) |
| `summary` | string | Краткое описание (обязательно) |
| `simple_explanation` | string | Простое объяснение (обязательно) |
| `examples` | array | Примеры (обязательно) |
| `common_mistakes` | array | Типичные ошибки (обязательно) |
| `review_questions` | array | Вопросы для проверки (обязательно) |

### Опциональные поля

| Поле | Тип | Описание |
|------|-----|---------|
| `tags` | array | Дополнительные теги |
| `next_review` | date | Когда повторить |
| `notes` | string | Дополнительные заметки |

## Процесс обработки SYNC_PACKAGE

### Шаг 1: Парсинг пакета
```python
# Агент извлекает все поля из пакета
# Проверяет наличие обязательных полей
# Валидирует значения
```

### Шаг 2: Проверка существования
```
Если заметка существует:
  → Дополнить содержимое
  → Обновить YAML frontmatter
  → Добавить новые связи
Если новая заметка:
  → Создать по шаблону Concept Template
  → Заполнить все секции
```

### Шаг 3: Создание связей
```
Для каждой заметки в related:
  Если не существует:
    → Создать stub-заметку
    → Добавить обратную ссылку на основную концепцию
  Если существует:
    → Добавить ссылку на основную концепцию
```

### Шаг 4: Обновление индексов
```
Index файл ([area]/Index.md):
  → Добавить [[topic]] в соответствующий раздел
  → Обновить дату обновления
```

### Шаг 5: Обновление логов
```
02 Progress/Learning Log.md:
  → Добавить запись о синхронизации
  → Обновить статистику

01 Roadmap/Roadmap.md:
  → Отметить прогресс по фазам
  → Отметить learnt концепции
```

### Шаг 6: Вывод отчета
```
Файлы обновлены:
- [Список всех файлов]

Статистика:
- Новых заметок: X
- Обновлено заметок: Y
- Новых связей: Z
```

## Примеры использования

### Пример 1: Полный пакет

```
SYNC_PACKAGE:
topic: Convolutional Neural Networks (CNN)
area: neural-networks
status: learning
confidence: 0.65
related: [Convolutional Layer, Pooling, Neural Networks, Image Classification, PyTorch]
summary: Нейронные сети с операциями свёртки для обработки изображений
simple_explanation: CNN применяют небольшие фильтры к изображениям для выделения признаков, сохраняя пространственную структуру
examples:
  - LeNet-5 для классификации рукописных цифр
  - ResNet для классификации ImageNet
  - Использование предварительно обученного MobileNet для мобильных устройств
common_mistakes:
  - Неправильный размер ядра свёртки (filter size)
  - Забывают нормализовать входные данные
  - Переобучение на маленьком датасете без augmentation
review_questions:
  - Как работает операция свёртки?
  - В чём разница между Padding='same' и 'valid'?
  - Когда нужна Global Average Pooling вместо Flatten?
  - Как выбрать количество фильтров в слое?
```

### Пример 2: Обновление существующей концепции

```
SYNC_PACKAGE:
topic: Feature Scaling
area: ml-basics
status: learned
confidence: 0.90
related: [Normalization, Standardization, KNN, SVM, Decision Trees]
summary: Масштабирование признаков к единому диапазону для улучшения производительности алгоритмов
simple_explanation: Приведение всех признаков к сопоставимому масштабу, чтобы ни один не доминировал
examples:
  - MinMaxScaler для масштабирования к [0, 1]
  - StandardScaler для z-score нормализации
notes: Обновлено с новыми примерами из практики
```

### Пример 3: Минимальный пакет

```
SYNC_PACKAGE:
topic: Attention Mechanism
area: neural-networks
status: draft
confidence: 0.50
related: [Transformers, Self-Attention, Multi-Head Attention]
summary: Механизм для фокусировки на релевантных частях входных данных
simple_explanation: Модель учится взвешивать важность разных частей входа
examples:
  - Self-attention в Transformers
common_mistakes:
  - Непонимание разницы между Attention и Self-Attention
review_questions:
  - Как вычисляется attention weights?
```

## Обработка ошибок

### Если пакет неполный
```
❌ Ошибка: Отсутствует обязательное поле 'confidence'
✅ Решение: Агент просит пользователя дополнить пакет
```

### Если концепция существует
```
⚠️ Внимание: Заметка Feature Scaling уже существует
✅ Решение: Показать существующее содержимое и спросить:
   - Обновить (дополнить)?
   - Перезаписать (с подтверждением)?
   - Создать новую версию?
```

### Если неверная область
```
❌ Ошибка: 'astronomy' не в списке допустимых областей
✅ Решение: Предложить список допустимых областей
```

## Интеграция с другими компонентами

### Связь с Sync Package Format

SYNC_PACKAGE Protocol — текстовый интерфейс к структуре `Templates/Sync Package.md`:
- Текстовый ввод → парсинг → JSON структура
- JSON → создание заметок → обновление базы

### Связь с Concept Template

Созданная заметка следует структуре `Templates/Concept Template.md`:
- Простое объяснение ← из `simple_explanation`
- Примеры ← из `examples`
- Типичные ошибки ← из `common_mistakes`
- Связанные темы ← из `related`

### Связь с Review Template

После синхронизации пользователь может создать `Review Template` для проверки знаний:
- Вопросы из `review_questions` используются в проверке

## Лучшие практики

### ✅ Рекомендуется

1. **Использовать стандартные области знания**
   - Выбирать из списка предопределённых областей
   - Не создавать новые области без необходимости

2. **Полностью заполнять пакет**
   - Все обязательные поля
   - Достаточно примеров и ошибок
   - Релевантные связанные концепции

3. **Следовать структуре Concept Template**
   - Примеры должны быть практичными
   - Ошибки — реальные и частые
   - Вопросы — проверяющие понимание

4. **Указывать реальный confidence уровень**
   - 0.0-0.3 — "не знаю"
   - 0.3-0.6 — "понимаю частично"
   - 0.6-0.8 — "хорошо понимаю"
   - 0.8-1.0 — "очень хорошо понимаю"

### ❌ Не рекомендуется

1. **Оставлять пакет неполным**
   - Все поля должны быть заполнены
   - Пустые массивы вызывают ошибки

2. **Использовать слишком много связей**
   - Оптимально 3-7 связанных концепций
   - Слишком много связей затрудняют навигацию

3. **Копировать содержимое с ChatGPT дословно**
   - Обрабатывать и адаптировать текст
   - Добавлять практические примеры
   - Улучшать для конкретного контекста

4. **Конфликтовать с существующими заметками**
   - Проверять наличие заметки перед синхронизацией
   - Согласовывать информацию при конфликтах

## Автоматизация

### Текущее состояние (v1.0)

SYNC_PACKAGE Protocol работает **вручную** с поддержкой агента:
- Пользователь вставляет пакет
- Агент обрабатывает и создаёт файлы
- Результат проверяется пользователем

### Будущая автоматизация (v2.0)

Планируется автоматический парсер:
```python
def process_sync_package(text):
    package = parse_package(text)
    validate(package)
    create_concept(package)
    create_stubs(package.related)
    update_indexes(package)
    return report()
```
