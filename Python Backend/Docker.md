# Docker

Инструмент для контейнеризации приложений.

## Простое объяснение

Docker упаковывает приложение со всеми зависимостями в контейнер, который работает везде одинаково.

## Интуитивное объяснение

Как морской контейнер — загрузил приложение со всем необходимым, отправил, оно работает везде.

## Зачем это нужно

- Одинаковая среда на всех машинах
- Легко масштабировать
- Быстрое развёртывание
- Изоляция приложений

## Как это работает

### Основные концепции
- Images (образы)
- Containers (контейнеры)
- Dockerfile
- Docker Compose

## Пример

```dockerfile
FROM python:3.10
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["python", "main.py"]
```

## Типичные ошибки

- Большие образы
- Нет .dockerignore
- Запуск процессов от root
