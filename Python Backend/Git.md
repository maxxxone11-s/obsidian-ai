# Git

Система контроля версий для отслеживания изменений кода.

## Простое объяснение

Git позволяет отслеживать историю изменений кода и работать в команде.

## Интуитивное объяснение

Как система архивирования документов — можно вернуться к любой старой версии документа.

## Зачем это нужно

- История всех изменений
- Совместная работа в команде
- Откат к предыдущим версиям
- Ветвление для разработки

## Как это работает

### Основные команды
- commit — сохранение изменений
- push — отправка на сервер
- pull — получение изменений
- branch — создание веток
- merge — объединение веток

## Пример

```bash
git add .
git commit -m "Add new feature"
git push origin main
```

## Типичные ошибки

- Коммиты в main вместо ветки
- Непонятные сообщения коммитов
- Нет .gitignore
