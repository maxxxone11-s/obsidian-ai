# OOP (Object-Oriented Programming)

Объектно-ориентированный подход к проектированию кода.

## Простое объяснение

OOP — способ организации кода через объекты, которые содержат данные и методы.

## Интуитивное объяснение

Как синий печать дома — объект содержит все необходимое (структуру, двери, окна) в одном месте, можно копировать и модифицировать.

## Зачем это нужно

- Структурированный и модульный код
- Переиспользуемость кода
- Легче поддерживать большие проекты

## Как это работает

### Основные концепции
- Classes (классы)
- Objects (объекты)
- Inheritance (наследование)
- Polymorphism (полиморфизм)
- Encapsulation (инкапсуляция)

## Пример

```python
class Dog:
    def __init__(self, name):
        self.name = name
    
    def bark(self):
        return f"{self.name} says woof!"

dog = Dog("Rex")
```

## Типичные ошибки

- Неправильная иерархия наследования
- Нарушение принципа единственной ответственности
- Излишняя сложность дизайна
