# Redis

In-memory хранилище данных для кэширования и обмена сообщениями.

## Простое объяснение

Redis — это быстрое хранилище данных в памяти для временного хранения и кэширования.

## Интуитивное объяснение

Как записная книжка для быстрых заметок — Redis быстрый, но не вечный.

## Зачем это нужно

- Кэширование часто используемых данных
- Очереди задач
- Сессионное хранилище
- Быстрая работа с данными

## Как это работает

### Основные структуры данных
- Strings (строки)
- Lists (списки)
- Sets (множества)
- Hashes (хеши)
- Sorted Sets (отсортированные множества)

## Пример

```python
import redis

r = redis.Redis()
r.set("key", "value")
value = r.get("key")
```

## Типичные ошибки

- Забывают про expiration (истечение срока)
- Используют Redis как основную БД
- Нет persistence стратегии
