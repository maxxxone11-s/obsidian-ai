# Правила ведения базы знаний

Эта база знаний ведется как Obsidian Vault для обучения Python Backend и AI.

## Общие правила

- Язык всех заметок: русский.
- Формат всех заметок: Markdown.
- Для связей между заметками использовать Obsidian `[[wikilinks]]`.
- Каждая учебная заметка должна содержать YAML frontmatter.
- Не дублировать заметки: перед созданием новой заметки проверить, нет ли уже существующей заметки по этой теме.
- Если тема уже существует, обновлять существующую заметку, а не создавать копию.
- После добавления или изменения важных заметок обновлять релевантные индексы в `Indexes`.
- После каждого учебного шага обновлять Learning Log в `02 Progress`.

## YAML frontmatter

Минимальный frontmatter для учебной заметки:

```yaml
---
title: Название темы
status: needs_review
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags:
  - python
  - backend
---
```

## Статусы

- Если тема понята не до конца, ставить `status: needs_review`.
- Если тема закреплена и может быть применена на практике, ставить `status: learned`.
- Статус должен отражать реальное понимание темы, а не факт прочтения материала.

## Индексы

- Индексы хранить в папке `Indexes`.
- Индексы должны ссылаться на ключевые заметки через `[[wikilinks]]`.
- Для каждой крупной области желательно поддерживать отдельный индекс:
  - `[[Python Backend Index]]`
  - `[[Machine Learning Index]]`
  - `[[Neural Networks Index]]`
  - `[[PyTorch Index]]`
  - `[[LangGraph Index]]`
  - `[[RAG Index]]`
  - `[[LLM Engineering Index]]`
  - `[[AI Agents Index]]`

## Learning Log

- Learning Log хранить в `02 Progress`.
- Записывать туда дату, изученную тему, ссылки на заметки, текущий статус и следующий шаг.
- Если после занятия появились пробелы в понимании, явно отметить их и связать с заметками со статусом `needs_review`.

## Inbox

- Быстрые идеи, вопросы и необработанные материалы сначала складывать в `Inbox`.
- Регулярно разбирать `Inbox`: переносить материалы в тематические папки, добавлять frontmatter и обновлять индексы.
