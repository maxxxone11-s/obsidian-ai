# LLM Tool Calling

## Простое объяснение
Модель принимает решение, а ToolNode выполняет действие.

## Зачем это нужно
Это основа современных AI-агентов, которые могут не только отвечать текстом, но и выполнять действия.

## Как это работает
LLM получает список доступных инструментов, выбирает подходящий вызов, а [[Tool Node]] исполняет его и возвращает наблюдение обратно в цикл агента.

## Пример
`tools_condition` направляет граф в `ToolNode`, если модель запросила инструмент.

## Типичные ошибки
- Думать, что Tool общается с пользователем.

## Вопросы для проверки
- Кто решает, какой Tool вызвать?
