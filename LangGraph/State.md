# State

Состояние графа в LangGraph.

## Простое объяснение

State — это данные, которые передаются между узлами графа в LangGraph.

## Интуитивное объяснение

Как контекст в разговоре — информация, которую нужно помнить и передавать между этапами.

## Зачем это нужно

- Управление данными в workflow
- Передача информации между узлами
- Отслеживание прогресса

## Как это работает

### Структура
- Может быть простой переменной или сложным объектом
- Обновляется при переходе между узлами

## Пример

```python
from langgraph.graph import StateGraph
from typing import TypedDict

class AgentState(TypedDict):
    messages: list
    context: str

graph = StateGraph(AgentState)
```

## Типичные ошибки

- Неправильная структура State
- Мутирование State без необходимости
- Забывают про типизацию

## Импорт KNOWLEDGE_EXPORT — 2026-06-26

**Кратко:** Общий объект данных, проходящий через весь граф.

**Простое объяснение:** Все [[Node]] читают и обновляют один общий State.

**Зачем это нужно:** Благодаря State узлы не зависят друг от друга напрямую.

**Как это работает:** State задаёт схему данных графа. Каждый Node получает текущий State и возвращает изменения, а LangGraph объединяет их с текущим состоянием.

**Пример:** State может содержать `user_input`, `ingredients`, `mode` и `final_answer`.

**Типичные ошибки:**
- Передавать данные напрямую между Node.
- Изменять State вручную вместо возврата изменений.

**Вопросы для проверки:**
- Почему Node работают через State?
