# Router

Маршрутизатор для выбора следующего узла в зависимости от State.

## Интуитивное объяснение

Как развилка на дороге — выбираешь путь в зависимости от ситуации.

## Кратко

Выбирает следующий маршрут на основе State.

## Простое объяснение

Router выбирает, к какому узлу перейти на основе условия на State.

Router не делает работу, а возвращает название следующего пути.

## Зачем это нужно

- Динамическая логика workflow
- Условный выбор пути
- Более гибкие графы

Router управляет всей логикой ветвлений.

## Как это работает

### Механизм
- Функция, возвращающая имя следующего узла
- Анализирует текущий State
- Определяет маршрут

Router читает [[State]], принимает решение и возвращает ключ маршрута, который описан в [[Conditional Edge]].

## Пример

```python
def route_decision(state):
    if state["question"]:
        return "answer_node"
    else:
        return "clarify_node"

graph.add_conditional_edges("router_node", route_decision)
```

`route_after_mode()` возвращает `diet`, `fried` или `balance`.

## Типичные ошибки

- Забывают проверить все возможные пути
- Неправильные названия узлов в возвращаемом значении
- Бесконечные циклы

- Вызывать Node внутри Router.
- Возвращать результат функции вместо имени маршрута.
