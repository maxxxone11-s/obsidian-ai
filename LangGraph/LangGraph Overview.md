# LangGraph Overview

## Простое объяснение

Агент описывается как набор шагов ([[Node]]) и переходов ([[Edge]]), которые работают через общий [[State]].

## Зачем это нужно

Позволяет строить сложных AI-агентов вместо одного большого цикла с LLM.

## Как это работает

LangGraph описывает workflow как граф: [[State]] хранит данные, [[Node]] выполняют шаги, [[Edge]] и [[Conditional Edge]] задают маршрут, а [[StateGraph]] собирает это в исполняемую структуру.

## Пример

WhatToCook: граф получает ввод пользователя, извлекает ингредиенты, выбирает режим приготовления и формирует рецепт.

## Типичные ошибки

- Думать, что LangGraph — это просто библиотека для вызова LLM.
- Считать Agent = LLM.
