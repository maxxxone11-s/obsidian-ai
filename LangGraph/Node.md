# Node

Узел в графе LangGraph, выполняющий задачу.

## Простое объяснение

Node — это функция, которая выполняет задачу и обновляет State.

## Интуитивное объяснение

Как рабочая станция на конвейере — выполняет свою часть работы и передаёт дальше.

## Зачем это нужно

- Разделение логики на отдельные задачи
- Модульность кода
- Переиспользование компонентов

## Как это работает

### Функция узла
- Принимает State
- Обновляет State
- Возвращает новый State

## Пример

```python
def process_node(state):
    state["messages"].append("processed")
    return state

graph.add_node("process", process_node)
```

## Типичные ошибки

- Неправильное обновление State
- Забывают вернуть обновленный State
- Мутирование глобального состояния

**Кратко:** Отдельный шаг выполнения графа.

**Простое объяснение:** Один Node = одна задача.

**Зачем это нужно:** Хорошая декомпозиция делает граф масштабируемым и понятным.

**Как это работает:** Node принимает [[State]], выполняет небольшую часть работы и возвращает [[State Update]].

**Пример:** `extract_ingredients()` извлекает ингредиенты из пользовательского запроса.

**Типичные ошибки:**
- Делать всю логику агента в одном Node.

**Вопросы для проверки:**
- Что должен делать хороший Node?
