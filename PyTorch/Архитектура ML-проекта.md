# Архитектура ML-проекта

## Простое объяснение

Структура проекта показывает архитектуру еще до чтения кода. Уже по названиям файлов можно определить поток данных и основные этапы работы приложения.

## Зачем это нужно

Разделение логики позволяет быстрее находить нужные части системы и менять части проекта без изменения остальных файлов.

## Как это работает

Типичная структура:

```text
model.py -> train.py -> model.pth -> predict.py
```

Dataset и Config используются во время обучения, но не должны смешиваться с архитектурой модели.
По наличию `predict.py` можно понять, что проект поддерживает inference.

## Пример

```text
project/
model.py
dataset.py
train.py
predict.py
config.py
utils.py
```

## Типичные ошибки

- Помещать всю логику проекта в один файл.
- Смешивать обучение, загрузку данных и архитектуру модели.
- Игнорировать структуру каталогов.
