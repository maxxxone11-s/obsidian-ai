# Separation of Concerns

## Простое объяснение

Один файл — одна ответственность.

## Зачем это нужно

Разделение ответственности позволяет менять одну часть проекта без изменения остальных.

## Как это работает

- `model.py` — модель.
- `dataset.py` — данные.
- `train.py` — обучение.
- `predict.py` — использование модели.

## Пример

Замена `dataset.py` не требует изменения `model.py`, если сохраняется тот же интерфейс данных.

## Типичные ошибки

- Объединять несколько ответственностей в одном файле.
- Связывать модель с конкретным источником данных.
