# Broadcasting

Механизм для операций между тензорами разных размеров.

## Простое объяснение

Broadcasting позволяет выполнять операции между тензорами с совместимыми, но не идентичными размерами.

## Интуитивное объяснение

Как транслирование сигнала — маленький сигнал повторяют, чтобы он подошел по размеру.

## Зачем это нужно

- Удобство при работе с данными
- Избежать явного копирования
- Более читаемый код

## Как это работает

### Правила broadcasting
1. Если shapes разные длины, дополнить единицами слева
2. Размеры совместимы, если они равны или один из них 1
3. Результат — максимальный размер в каждой размерности

## Пример

```python
import torch

x = torch.randn(3, 1)    # shape: (3, 1)
y = torch.randn(1, 4)    # shape: (1, 4)
z = x + y                # результат: (3, 4)
```

## Типичные ошибки

- Неправильное предположение о broadcasting
- Не проверяют shapes перед операцией
- Путаница с порядком размеров
