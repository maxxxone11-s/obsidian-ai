# Broadcasting

Broadcasting позволяет выполнять операции между Tensor разной формы без явного копирования данных.

## Простое объяснение

PyTorch логически расширяет Tensor, если это возможно. Он не копирует данные физически, а подставляет меньший Tensor так, будто он подходит по форме.

## Зачем это нужно

- Broadcasting используется почти во всех вычислениях нейросетей.
- Он делает код короче и избавляет от ручного повторения данных.
- Он важен для нормализации, batch operations, attention и работы с logits.

## Как это работает

Правила сравниваются справа налево:

1. Размерности должны совпадать.
2. Или одна из размерностей должна быть равна `1`.
3. Если правило нарушено, операция невозможна.

Главный принцип: Broadcasting никогда не угадывает. Он работает только там, где расширение однозначно.

## Пример

```python
import torch

x = torch.randn(8, 5)
y = torch.randn(5)

z = x + y
print(z.shape)  # torch.Size([8, 5])
```

Мысленно PyTorch читает это как:

```text
(8, 5)
+
(1, 5)
=
(8, 5)
```

А вот так не сработает:

```text
(8, 5)
+
(8,)
```

Потому что справа сравниваются `5` и `8`, а они не равны и ни одно из них не `1`.

## Типичные ошибки

- Думать, что Tensor можно растянуть, если размер делится без остатка.
- Путать реальное поведение PyTorch с мысленным представлением `(1, n)`.
- Забывать, что сравнение размерностей идёт справа налево.
- Не проверять shapes перед операцией.
