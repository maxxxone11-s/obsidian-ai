# Dataset

Stub-заметка для PyTorch Dataset.

## Простое объяснение

Dataset описывает, как получить один объект обучения. В проекте `dataset.py` отвечает только за предоставление данных модели.
Если Dataset продолжает возвращать данные в прежнем формате, модель не требует изменений.

## Зачем это нужно

- Хранит логику доступа к данным.
- Работает вместе с [[DataLoader]].
- Возвращает пары вроде `(features, label)`.
- Позволяет менять источник данных без изменения обучения и модели.
- Позволяет масштабировать проект и менять инфраструктуру без изменения логики обучения.

## Как это работает

Обычно Dataset реализует методы `__len__` и `__getitem__`. Он должен гарантировать формат выходных данных, например `(features, label)`, который затем забирает [[DataLoader]].
Источник Tensor может измениться, но интерфейс должен остаться прежним.

## Пример

```python
from torch.utils.data import Dataset

class MyDataset(Dataset):
    def __len__(self):
        return 0

    def __getitem__(self, index):
        raise NotImplementedError
```

## Типичные ошибки

- Смешивать Dataset и DataLoader.
- Возвращать данные неправильного dtype или shape.
- Делать тяжелую подготовку данных в неподходящем месте.
- Помещать обучение модели внутрь Dataset.
- Считать Dataset местом хранения модели.
- Встраивать чтение файлов внутрь модели.
