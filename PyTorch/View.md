# View

Изменение формы тензора без копирования данных.

## Простое объяснение

View изменяет форму тензора, но данные остаются на месте в памяти.

## Интуитивное объяснение

Как смотреть на матрицу с разных углов — данные те же, но вид разный.

## Зачем это нужно

- Эффективное изменение формы
- Подготовка данных для слоёв
- Избежать копирования памяти

## Как это работает

### Требования к view
- Тензор должен быть contiguous
- Общее количество элементов не меняется

## Пример

```python
import torch

x = torch.randn(6)
y = x.view(2, 3)  # shape: (2, 3)
z = y.view(-1)    # -1 означает infer (6,)
```

## Типичные ошибки

- View non-contiguous tensors (ошибка)
- Забывают про -1 для автоматического расчёта
- Путаница с transpose и view
