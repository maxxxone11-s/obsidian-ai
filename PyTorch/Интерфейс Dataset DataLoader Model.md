# Интерфейс Dataset DataLoader Model

## Простое объяснение

Каждая часть знает только формат входных и выходных данных следующей части.
Для взаимодействия компонентов важен интерфейс, а не внутренняя реализация.

## Зачем это нужно

Так можно заменить Dataset без изменения Model и Training Loop.
Это уменьшает связанность компонентов и позволяет менять источник данных без изменения остального проекта.

## Как это работает

```text
Dataset
↓
(features, label)
↓
DataLoader
↓
batch_x, batch_y
↓
Model
↓
prediction
```

## Пример

Dataset может читать JPEG, PNG, PostgreSQL или Amazon S3. Model останется неизменной, если получает данные в ожидаемом формате.
`train.py` использует одинаковый training loop, пока получает `(features, label)`.

## Типичные ошибки

- Связывать модель с конкретным источником данных.
- Заставлять `train.py` знать детали чтения файлов или базы данных.
- Делать остальные модули зависимыми от реализации Dataset.
