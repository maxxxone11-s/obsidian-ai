# Activation Function

Функция активации для введения нелинейности.

## Интуитивное объяснение

Как включатель света — пока не достигнешь порога, ничего не происходит, потом резко включается.

## Кратко

Activation Function преобразует Score в Output.

## Простое объяснение

Activation Function преобразует сумму взвешенных входов, добавляя нелинейность.

Она решает, каким станет итоговый сигнал нейрона.

## Зачем это нужно

- Добавляет нелинейность
- Позволяет сети учиться сложным функциям
- Без неё сеть просто линейная модель

Без функций активации глубокие нейросети были бы практически бесполезны.

## Как это работает

### Основные функции
- ReLU (Rectified Linear Unit)
- Sigmoid
- Tanh
- Softmax

### Свойства
- Должны быть дифференцируемыми
- Часто ограничены в диапазоне

Функция активации добавляет нелинейность: берёт [[Score]] и возвращает [[Output]]. Благодаря этому несколько слоёв могут моделировать сложные зависимости.

## Пример

```python
import torch.nn as nn

relu = nn.ReLU()
x = torch.tensor([-1.0, 0.0, 1.0])
print(relu(x))  # tensor([0., 0., 1.])
```

Score → ReLU → Output.

## Типичные ошибки

- Используют sigmoid везде (dying sigmoid)
- Забывают про activation function
- Неправильно выбирают для задачи

- Считать, что функция активации обучается.
